# 숏폼 자동 생성 시스템 - 프롬프트 가이드

## 시스템 개요
짧게 촬영된 여러 개의 영상 클립을 자동으로 조합하여 15초 내외의 숏폼 영상을 생성하는 시스템.
완성된 영상에는 자막과 배경음악이 자동으로 포함된다.

## 대상 플랫폼
- 인스타그램 릴스
- 유튜브 쇼츠
- 틱톡
- 공통 해상도: 1080 x 1920 (9:16 세로)

---

## 폴더 구조

```
shortform_project/
├── prompt.md                  ← 이 파일 (작업 지시서)
├── raw_clips/                 ← 촬영 원본 클립 (로컬 또는 Google Drive)
│   └── (아래 '영상 소스 경로' 참고)
├── bgm/                       ← 배경음악 (로컬 또는 Google Drive)
│   └── (아래 'BGM 소스 경로' 참고)
├── fonts/                     ← 자막용 한글 폰트 파일 (.ttf)
│   └── NanumSquareB.ttf
├── output/                    ← 완성된 숏폼 영상 저장 (1개 파일)
├── temp/                      ← 작업 중 임시 파일
└── scripts/                   ← 자동화 스크립트 저장
    ├── 01_analyze_clips.py    ← 클립 분석 (길이, 해상도 등)
    ├── 02_combine_clips.py    ← 클립 조합 + 트리밍 (15초)
    ├── 03_generate_subs.py    ← 음성 인식 → 자막 생성
    ├── 04_add_subs_bgm.py     ← 자막 + BGM 합성
    └── 05_export_all.py       ← 플랫폼별 최종 출력
```

## 영상 소스 경로

영상 클립은 아래 경로에서 직접 읽어온다. 로컬 복사 없이 Google Drive 마운트 경로를 그대로 사용한다.

- **기본 경로 (Google Drive):**
  ```
  ~/Library/CloudStorage/GoogleDrive-tkfkdgowldms@gmail.com/내 드라이브/
  ```
- 사용자가 특정 폴더를 지정하면 해당 경로 사용 (예: "0. 위즈더플래닝 촬영본 폴더에서 가져와")
- `raw_clips/` 폴더에 직접 파일이 있으면 그것을 우선 사용
- Google Drive 경로의 파일은 복사하지 않고 직접 참조하여 용량 절약

## BGM 소스 경로

BGM은 아래 Google Drive 경로에서 직접 읽어온다. 로컬 복사 없이 마운트 경로를 그대로 사용한다.

- **기본 경로 (Google Drive):**
  ```
  ~/Library/CloudStorage/GoogleDrive-tkfkdgowldms@gmail.com/내 드라이브/3.위즈더플래닝 디자인/숏폼/BGM/
  ```
- **장르별 하위 폴더 구조:**
  - `강렬한/` ← 파워풀, 임팩트 (2곡)
  - `밝음/` ← 밝고 따뜻한 (7곡)
  - `신남/` ← 업비트, 에너지 (99곡) ★기본값
  - `일본풍/` ← 일본 느낌 (3곡)
  - `잔잔/` ← 차분, 부드러운 (37곡)
  - `크리스마스/` ← 시즌 특화 (12곡)
  - `클래식/` ← 클래식 (7곡)
  - `팝/` ← 팝 (3곡)
  - `펑키/` ← 그루비, 리듬감 (8곡)
- 로컬 `bgm/` 폴더에 파일이 있으면 그것을 우선 사용

## 작업 워크플로우

### Step 1: 클립 분석
- 영상 소스 경로(Google Drive 또는 `raw_clips/`)의 모든 영상 파일을 스캔
- 각 클립의 길이(초), 해상도, 코덱 정보를 분석
- 파일명 앞의 숫자(01_, 02_, ...)를 기준으로 순서 정렬
- 분석 결과를 터미널에 표로 출력
- (중요) `pip install python-dotenv google-genai` 명령어로 환경설정을 확실히 하고, 루트 폴더의 `.env`에 `GEMINI_API_KEY=...`가 있는지 체크

### Step 2: 클립 조합 및 트리밍
- 정렬된 순서대로 클립을 연결
- 전체 합산 길이가 15초를 초과하면:
  - 각 클립을 균등 비율로 트리밍 (앞뒤 잘라내기)
  - 또는 마지막 클립에서 초과분 제거
- 전체 합산 길이가 15초 미만이면:
  - 마지막 클립 끝에 페이드아웃 적용하여 자연스럽게 마무리
- 클립 간 전환: 0.3초 크로스페이드 적용
- 해상도 통일: 모든 클립을 1080x1920 (9:16)으로 리사이즈
  - 가로 영상인 경우 → 세로 프레임에 맞추고 상하 블러 배경 추가

### Step 3: 자막 생성 (Gemini 멀티모달 오디오/비전 통합 분석)
- **사용자 제공 자막 우선**: 사용자가 대본이나 자막(텍스트/SRT)을 제공하는 경우, 해당 내용을 그대로 자막으로 사용한다 (AI 분석은 건너뜀).
- (제공된 자막이 없을 경우) **Google Gemini 1.5 Flash API**를 사용하여 영상 전체(화면+소리)를 분석:
  - 영상을 Gemini에 업로드하여 "이 영상의 내용에 어울리는 짧고 임팩트 있는 숏폼용 한글 SRT 자막을 만들어줘"라고 프롬프트 전달.
  - 무음 영상이거나 배경 소음만 있는 경우(예: 고기 굽는 소리, 무음 스포츠), 엉뚱한 대화 자막(할루시네이션)을 만들지 않고 **"지글지글 맛있게 익어가는 고기!", "나이스 샷!" 등 맥락에 맞는 감성적인 나레이션 형태의 자막**을 생성하도록 지시.
  - 유의미한 한국어 음성이 있는 경우, 해당 음성을 살리되 숏폼에 맞게 요약/강조되게 정리.
- 생성된 텍스트를 SRT 파일 형식으로 저장.

### Step 3.5: BGM 자동 선택
사용자가 BGM 장르를 직접 지정하지 않은 경우, 아래 정보를 종합하여 적절한 장르를 자동 선택한다.

- **판단 입력:**
  1. 영상 소스 경로의 파일명에 포함된 키워드
  2. Step 3에서 생성된 자막(SRT) 텍스트 내용
  3. 영상의 전반적 분위기 (말투, 감탄사 빈도, 내용의 톤)

- **장르 매핑 기준:**

  | 영상 분위기 | 선택 장르 |
  |---|---|
  | 에너지 넘치는, 빠른 템포, 챌린지 | 신남 |
  | 극적인, 강한 임팩트, 하이라이트 | 강렬한 |
  | 따뜻한, 일상적, 브이로그, 카페 | 밝음 |
  | 감성적, 힐링, 풍경, 여행 | 잔잔 |
  | 유쾌한, 리뷰, 언박싱, 먹방 | 펑키 |
  | 고급스러운, 교육적, 격식있는 | 클래식 |
  | 트렌디한, 노래, 커버 | 팝 |
  | 일본 관련 콘텐츠 (도쿄, 오사카 등) | 일본풍 |
  | 크리스마스, 연말, 겨울 시즌 | 크리스마스 |
  | 무음 영상 (자막 없음) | 잔잔 |
  | 판단 불가 | 신남 (기본값) |

- **선택 규칙:**
  - 사용자가 직접 장르를 지정하면 → 해당 장르 최우선
  - BGM 소스 경로의 해당 장르 폴더에서 랜덤으로 1곡 선택
  - 선택한 장르와 곡명을 터미널에 출력: `🎵 BGM 선택: [장르] - [파일명]`

### Step 4: 자막 + BGM 합성
- **자막 스타일 (캡컷 스타일 모션 텍스트):**
  - **폰트:** AppleSDGothicNeo.ttc (Bold=index6, SemiBold=index4, Medium=index2, Regular=index0)
  - 제목/강조: Bold, 본문: SemiBold, 부제: Medium 등 문맥에 따라 굵기를 다르게 사용
  - **폰트 크기:** 105~220pt (기본 설정 대비 1.5~2배 확대, 매우 크고 임팩트 있게). 핵심 키워드는 더 크게(200pt+)
  - **색상 규칙:**
    - 기본 글자: 흰색 (#FFFFFF)
    - 업체명/핵심 키워드: 시안(#00FFFF), 노란(#FFD700), 초록(#00FF80) 등 강조색
    - CTA(행동유도) 문구: 빨강/주황 계열
    - 테두리: 검정색 5px 외곽선 + 그림자
  - **위치:** 문구마다 다르게 배치 (상단/중앙/하단 번갈아)
  - **내용 밀도:** 짧은 문구를 빠르게 교체 (1.5~2.5초 단위). 15초 기준 7~10개 문구
  - **모션 효과 (프레임 단위 애니메이션):**
    - 팝업(pop-in): 0에서 커지며 등장
    - 슬라이드업/다운: 아래/위에서 등장
    - 바운스: 살짝 튀는 효과
    - 스케일 펄스: 등장 후 살짝 커졌다 작아짐
    - 문구마다 다른 효과를 번갈아 적용
  - Python Pillow로 프레임별 PNG 시퀀스를 생성하여 ffmpeg overlay로 합성
- **BGM 설정:**
  - Step 3.5에서 자동 선택된 BGM 파일 사용
  - 원본 음성 볼륨: 100%
  - BGM 볼륨: 원본 대비 40~50% (확실히 들리는 수준)
  - BGM은 영상 길이에 맞춰 자동 페이드인(0.2초) / 페이드아웃(1초)

### Step 5: 최종 출력
- 출력 사양:
  - 코덱: H.264
  - 오디오: AAC 128kbps
  - 프레임레이트: 30fps
  - 해상도: 1080x1920 (9:16 세로)
- 출력 파일 1개만 생성 (릴스/쇼츠/틱톡 공통 사양)
- 파일명 규칙: `shortform_YYYYMMDD_업체명.mp4`
- 저장 위치: `output/` 폴더

## 주의사항

1. 영상 소스 경로(raw_clips/ 또는 Google Drive)에 영상 파일이 최소 1개 이상 있어야 실행 가능
2. 파일명 앞에 숫자를 붙여 순서를 지정할 것 (01_, 02_, 03_...)
3. 숫자가 없으면 파일명 알파벳 순으로 자동 정렬
4. 지원 포맷: .mp4, .mov, .avi, .mkv
5. 한글 폰트 파일이 없으면 자막이 깨질 수 있음 → fonts/ 폴더에 .ttf 파일 필수
6. BGM 파일이 없으면 자막만 입힌 버전으로 출력
7. Google Drive 연결이 안 되면 로컬 bgm/ 폴더를 폴백으로 사용
